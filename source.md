# AI Frontend Developer Job Plan (React.js / Next.js)

## 1) Target role
**Goal title:** AI Frontend Developer / Frontend Engineer (AI Products)

### What hiring managers want
- Strong React.js + Next.js fundamentals.
- Ability to integrate AI APIs (OpenAI, Anthropic, Azure OpenAI, local models).
- Production UI quality: accessibility, performance, responsive design.
- Experience with real-time UX patterns (streaming responses, optimistic UI, retries).
- Understanding of prompt design and model limitations.

## 2) Portfolio projects to build (must-have)

### Project A: AI Chat Workspace (Next.js)
- Authentication (Clerk/Auth.js/Firebase).
- Streaming chat responses.
- Conversation history with search.
- Markdown rendering with code blocks.
- Cost/token usage display.

### Project B: AI Content Studio (React + Node)
- Generate social posts, emails, and summaries.
- Prompt templates by use case.
- Editable output and version history.
- Export to PDF/Markdown.

### Project C: AI Support Dashboard (Next.js + charts)
- Ticket triage with AI suggestions.
- Sentiment and urgency tagging.
- Human-in-the-loop approval flow.
- Analytics panel for response quality.

## 3) Skills checklist (interview-ready)
- React: hooks, context, state management, reusable components.
- Next.js: App Router, SSR/SSG/ISR, API routes/server actions.
- TypeScript for safer frontend architecture.
- Styling: Tailwind CSS + component systems.
- Testing: Jest + React Testing Library + Playwright basics.
- AI integration: prompt patterns, rate limits, retries, fallback UX.
- Performance: lazy loading, memoization, image optimization, Web Vitals.

## 4) Resume bullets (copy-ready)
- Built and deployed AI-powered frontend apps using **React.js/Next.js** with streaming LLM responses and responsive UX.
- Integrated **OpenAI-compatible APIs** with token usage tracking, retry handling, and error-resilient state flows.
- Improved page performance using code splitting and optimized rendering strategies, raising Lighthouse scores.
- Shipped reusable UI components and maintained high accessibility standards across dashboard workflows.

### Week 4
